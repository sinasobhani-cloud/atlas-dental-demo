The Open Dental data chain every endpoint verified against the live API

One Open Dental database per location, one API key per office. We poll (Open Dental has no webhooks) using timestamp cursors. The agent's entire "context pack" for a verification call is assembled from data the practice already maintains — including which phone number to dial:
appointments (T+1..T+3) → AptNum, PatNum, AptDateTime, ProvNum └─ patients/{PatNum} → name, DOB, phone └─ procedurelogs?AptNum → planned procedure codes → question template └─ providers/{ProvNum} → NationalProvID (NPI) └─ patplans?PatNum → primary/secondary → InsSubNum └─ inssubs/{InsSubNum} → SubscriberID ← the Medicaid member ID └─ insplans → CarrierNum, GroupNum └─ carriers → CarrierName, Phone ← number we dial, ElectID ← EDI payer ID
Writes are minimal by design: the only thing the platform ever writes back is a new appointment when a patient books — everything else is read-only. Conversion detection (lead → booked) works by watching for new appointments whose phone matches a lead.

Verification waterfall cheapest channel first — the AI call is the last resort, not the headline

1 · TMHP 270/271 (EDI)Eligible this month? Which DMO (MCNA / DentaQuest / UHC)? Seconds, structured, near-free. Resolves most checks.
↓ gaps remain
2 · DMO portal automationBenefits, frequency limits, service history, prior-auth flags — richer than the 271.
↓ still missing fields
3a · AI call — TMHP AIS IVR (800-925-9126)Pure keypad navigation, no human on the line. The safe proving ground for the calling stack.
↓ benefits detail needed
3b · AI call — DMO provider lineIVR + hold (agent eats this), then rep conversation per the decision tree. Human takeover one click away at all times.
↓
Output · VerificationRecordPer-code coverage, frequencies, history dates, PA flags, reference number — published to tomorrow's schedule, or routed to the review queue if confidence is low.

The autonomy ladder per payer — promoted only on measured accuracy

LevelAgent doesHuman doesPromotion gate
L1 · Hold-for-meDials, navigates IVR, enters NPI/member ID, survives hold; alerts human the instant a rep answersThe 3-minute rep conversationStarting level for every payer — rep never talks to an AI
L2 · SupervisedFull rep conversation, live-monitoredWatches console (1 : 5–10 calls), one-click barge-in≥95% extraction accuracy over 50 shadowed calls
L3 · AutonomousEverythingReview queue only≥98% accuracy over 100 calls, takeover <5%
Full detail — call state machine (~40 states), takeover triggers, hard rules, fine-tuning loop — lives in the repo: docs/verification-call-playbook.md and docs/agent-decision-tree.md. The front-desk call-logging sheet (docs/call-logging-sheet.md) turns 2 weeks of real calls into the payer scripts.
